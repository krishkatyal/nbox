NBX-Build Instances#

NBX-Build Instances are APIs to your machines. These APIs can be used to change state of the machine (start, stop, etc.), can be used to transfer files to and from the machine and to an extent running and managing programs (WIP).

Bases: object

NBX-Build Instance class manages the both individual instance, but provides webserver functionality using nbox_ws_v1, such as starting and stopping, deletion and more.

Parameters

  • i (str) – name or project_id of the instance

  • workspace_id (int, optional) – id of the workspace to use, if not provided personal workspace will be used.

  • cs_endpoint (str, optional) – endpoint to use for the webserver, this will connect to the custom ports functionality of the instance. Defaults to server,

start(cpu: int = 2, gpu: str = 'p100', gpu_count: int = 0, auto_shutdown: int = 6, dedicated_hw: bool = False, zone='asia-south-1', *, _ssh: bool = False)[source]#

Start instance if not already running and loads APIs from the compute server.

Actual start is implemented in _start method, this combines other things

Parameters

  • cpu (int, optional) – CPU count should be one of [2, 4, 8]

  • gpu (str, optional) – GPU name should be one of ["t5", "p100", "v100", "k80"]

  • gpu_count (int, optional) – When zero, cpu-only instance is started

  • auto_shutdown (int, optional) – No autoshutdown if zero, defaults to 6.

  • dedicated_hw (bool, optional) – If not spot/pre-emptible like machines used

  • zone (str, optional) – GCP cloud regions, defaults to “asia-south-1”.
